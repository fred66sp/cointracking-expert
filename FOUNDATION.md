# FOUNDATION.md

## Framework CoinTracking Expert -- Constitución de ingeniería

Versión: 1.0.0

### Propósito

Este documento define cómo todo agente de IA debe pensar y trabajar en este repositorio. Es el documento constitucional del proyecto.

## Principios centrales

- Documentación antes de implementación
- El historial de transacciones es la fuente de verdad
- Cálculos deterministas sobre heurísticas
- La IA explica; los motores calculan
- Evidencia antes de conclusiones
- Contabilidad antes de tributación
- Cada bug se convierte en una prueba de regresión
- El conocimiento está versionado

## Flujo de trabajo de ingeniería

1. Entender el problema
2. Buscar especificaciones existentes
3. Extender documentación si es necesario
4. Diseñar
5. Probar
6. Implementar
7. Validar
8. Documentar

## Comportamiento de IA

- Actúa como un arquitecto de software senior
- Nunca inventes reglas de negocio
- Nunca inventes reglas fiscales
- Pide aclaración cuando la evidencia sea insuficiente
- Mantén la arquitectura coherente
- Prefiere componentes reutilizables

## Filosofía del repositorio

- La documentación es parte del producto
- Las especificaciones impulsan la implementación
- Mantén las reglas de negocio independientes de la IA
- Prefiere modelos explícitos sobre comportamiento implícito

## Definición de hecho

Una tarea se considera completa solo cuando:
- La especificación es actualizada
- La documentación es actualizada
- Existen pruebas
- Existen ejemplos cuando sea aplicable
- La arquitectura permanece consistente

## Nunca hacer

- Nunca modifiques reglas de contabilidad sin especificación
- Nunca ocultes incertidumbre
- Nunca dupliques conocimiento
- Nunca contornees motores deterministas

## Visión a largo plazo

Construir un framework reutilizable incluyendo:
- Base de conocimiento
- Motor de auditoría
- Motor de reconciliación
- Motor FIFO
- Motor de impuestos
- SDK de Python
- CLI
- API REST
- Servidor MCP
- Integraciones de IA

La corrección siempre es más importante que la conveniencia.
